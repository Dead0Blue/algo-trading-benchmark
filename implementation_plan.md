# Algorithmic Trading & ML — Python vs Java

Build the **same algorithmic trading system** in both Python and Java, then benchmark and compare them across performance, accuracy, and developer experience.

---

## Project Overview

**Goal:** Create an ML-powered stock trading signal generator and backtester in **both languages**, using the **same dataset, features, and model types**, to produce a fair, apples-to-apples comparison.

**Stock tickers to use:** AAPL, MSFT, TSLA (configurable)
**Timeframe:** 5 years of daily OHLCV data

## Proposed Changes

### Phase 1 — Python Implementation

> Build the complete trading system in Python first, since it's faster to prototype.

#### [NEW] python/requirements.txt
- Dependencies: `yfinance`, `pandas`, `numpy`, `scikit-learn`, `matplotlib`, `ta-lib` (or `ta`)

#### [NEW] python/src/data_fetcher.py
- Use `yfinance` to download 5 years of daily OHLCV data for configurable tickers
- Save to CSV for reproducibility
- Handle missing data / market holidays

#### [NEW] python/src/features.py
- Compute technical indicators from raw OHLCV:
  - **RSI** (Relative Strength Index, 14-period)
  - **MACD** (12, 26, 9)
  - **Bollinger Bands** (20-period, 2 std dev)
  - **EMA** (Exponential Moving Average, 10 & 50 period)
  - **ATR** (Average True Range)
  - **Volume Change %**
- Create target label: `1` if next-day close > today's close, else `0`

#### [NEW] python/src/model.py
- Implement two models:
  1. **Random Forest Classifier** (scikit-learn)
  2. **LSTM Neural Network** (Keras/TensorFlow)
- Train/test split: 80/20 chronological (no random shuffle — time-series!)
- Output: accuracy, precision, recall, F1-score, confusion matrix

#### [NEW] python/src/backtest.py
- Simulate trades based on model predictions
- Starting capital: $10,000
- Calculate:
  - Total Return %
  - Sharpe Ratio
  - Max Drawdown
  - Win Rate
  - Number of trades
- Compare vs Buy-and-Hold benchmark
- Generate equity curve chart

#### [NEW] python/src/main.py
- Orchestrate the full pipeline: fetch → features → train → backtest
- Log execution time for each phase (for comparison with Java)

### Phase 2 — Java Implementation

> Re-implement the exact same logic in Java, module-for-module.

#### [NEW] java/pom.xml
- Maven project with dependencies:
  - **Smile** (ML algorithms — Random Forest)
  - **Deeplearning4j** (LSTM neural network)
  - **Tablesaw** (DataFrame-style data manipulation)
  - **JFreeChart** (plotting equity curves)
  - **OkHttp** or CSV parsing library (data loading)

#### [NEW] java/src/main/java/trading/data/DataFetcher.java
- Load the **same CSV files** generated by the Python script (ensures identical data)
- Parse into Tablesaw DataFrames

#### [NEW] java/src/main/java/trading/features/FeatureEngine.java
- Compute the **exact same** technical indicators: RSI, MACD, Bollinger Bands, EMA, ATR, Volume Change
- Same parameters, same formulas

#### [NEW] java/src/main/java/trading/model/TradingModel.java
- Implement:
  1. **Random Forest** using Smile library
  2. **LSTM** using Deeplearning4j
- Same train/test split (80/20 chronological)
- Same evaluation metrics

#### [NEW] java/src/main/java/trading/backtest/BacktestEngine.java
- Same backtesting logic: $10,000 starting capital
- Same metrics: Return, Sharpe, Drawdown, Win Rate
- Equity curve output via JFreeChart

#### [NEW] java/src/main/java/trading/Main.java
- Same pipeline with execution time logging per phase
